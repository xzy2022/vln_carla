以 UE5 为主，和你目标最相关的接口可以收敛成这组。

**1) 指定车型**
- 蓝图库入口：`world.get_blueprint_library()`  
  参考：`Docs_Carla_UE5/core_actors.md:31`，`Docs_Carla_UE5/python_api.md:3301`
- 选车型：
  - 精确：`blueprint_library.find("vehicle.xxx.yyy")`
  - 模糊：`blueprint_library.filter("vehicle.*")` 或 `filter("vehicle.bmw.*")`  
  参考：`Docs_Carla_UE5/core_actors.md:39`，`Docs_Carla_UE5/bp_library.md:655`
- 可选属性（颜色/driver_id/role_name）用 `set_attribute`。  
  参考示例：`PythonAPI_Carla_UE5/examples/manual_control.py:261`

**2) 指定位置生成车辆**
- 构造位姿：`carla.Transform(carla.Location(x,y,z), carla.Rotation(pitch,yaw,roll))`  
  参考：`Docs_Carla_UE5/python_api.md:1302`，`Docs_Carla_UE5/python_api.md:1650`，`Docs_Carla_UE5/python_api.md:2207`
- 生成：
  - `world.spawn_actor(...)`：失败抛异常
  - `world.try_spawn_actor(...)`：失败返回 `None`（更适合业务兜底）  
  参考：`Docs_Carla_UE5/core_actors.md:73`，`Docs_Carla_UE5/core_actors.md:74`，`Docs_Carla_UE5/python_api.md:3256`，`Docs_Carla_UE5/python_api.md:3271`
- 注意：
  - 该点若有碰撞不会生成  
  - `Rotation` 参数顺序是 `(pitch, yaw, roll)`，不是 UE Editor 的顺序  
  参考：`Docs_Carla_UE5/core_actors.md:82`，`Docs_Carla_UE5/core_actors.md:84`

**3) 简单运动学控制**
- 主控接口：`vehicle.apply_control(carla.VehicleControl(...))`  
  参考：`Docs_Carla_UE5/core_actors.md:219`，`Docs_Carla_UE5/python_api.md:2408`
- `VehicleControl` 关键范围：
  - `throttle` `[0,1]`
  - `steer` `[-1,1]`
  - `brake` `[0,1]`  
  参考：`Docs_Carla_UE5/python_api.md:2544`
- 如果你要更“运动学式”的速度控制（少关心油门/刹车动态）：
  - `set_target_velocity(...)`
  - `enable_constant_velocity(...)`（局部坐标速度）  
  参考：`Docs_Carla_UE5/python_api.md:136`，`Docs_Carla_UE5/python_api.md:87`
- 注意与 TM/autopilot 冲突：`set_autopilot(True)` 后由 Traffic Manager 管车。  
  参考：`Docs_Carla_UE5/python_api.md:2490`

**4) 控制循环建议（UE5）**
- 用同步模式+固定步长：`WorldSettings.synchronous_mode=True`，`fixed_delta_seconds=...`，每步 `world.tick()`。  
  参考：`Docs_Carla_UE5/python_api.md:3392`，`Docs_Carla_UE5/python_api.md:3189`
- 文档也提示大量异步 `set()` 会累积延迟。  
  参考：`Docs_Carla_UE5/core_actors.md:140`
